An In-Depth Look At Pointers

This explanation of pointers will be written in cpp. Let's try compiling:


#include <iostream>

int main() {
	int b = 10;
	int *a = &b;
	std::cout << "The address location of b = " << &b << std::endl;
	std::cout << "The content at the pointer a = "<< a << std::endl;
	std::cout << "a points to the content at b = "<< *a << std::endl;
	return 0;
}

Let's Learn About Memory

Ask permission from the computer to allocate memory space to store a datatype (in this case int). Name the allocated memory b.

int b;

By assigning a value to b, we store the integer 10 at the memory location we call b.

b = 10;

Ask the computer for permission to read the content at b. Next, add the content to the integer 1. Then, I am asking for the summation to be assigned to b. Notice that 1 is not a double like 1.0. Since b only has enough memory for integer values, adding other datatypes like std::string, double, or long int can cause unpredictable errors.

b = b + 1;

When you ask the computer to read at the memory location named b, you are accessing a numerical address in hexadecimal. To call the serial number location of your variable b use the ampersand symbol, &:

std::cout << "The address location of b = " << &b << std::endl;

To call the hexadecimal address in numbers only:

std::cout << (long int)(&b) << std::endl;

Introduction to Accessors and Mutators

An accessor only has access to or "gets" the variables. That's why accessors are oftentimes known as get functions. Thus, the variables remain constant. Let's compile the get function, getvar


#include <iostream>

int getvar(int content) {
	return content;
}

int main() {
	int b = 10;
	b = b + 1;

	std::cout << getvar(b) << std::endl;
	return 0;
}
  1. Request the memory for an integer and name it b.
  2. At that memory location or address assign the integer 10.
  3. Read the content at the address named b.
  4. Add 1 to the content at b.
  5. Write the content into the address of b.
  6. Set b as the argument for the function getvar
  7. Request the memory for an integer and name it content.
  8. Read the content in b and copy that content into the address named content.
  9. Return content
  10. Print the return value of the function getvar

On the other hand, mutators change or "set" the values of variables. That's why mutators are called setter functions. Let's try modifying getvar to a mutator function by changing the value in b.


#include <iostream>

int getvar(int content) {
	std::cout << "The location of content is at: " << (long int) &content << std::endl;
	content = content + 1;
	return content;
}

int main() {
	int b = 10;
	b = b + 1;

	std::cout << "The value of content is: " << getvar(b) << std::endl;
	std::cout << "The location of b is at: " << (long int) &b << std::endl;
	std::cout << "The value of b is: " << b << std::endl;
	return 0;
}
							
int* a = &b

Because the memory location of b is stored in a, the pointer a can also access the value b.
