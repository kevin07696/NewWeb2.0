Introduction to Pointers

This explanation of pointers will be written in cpp. Let's try compiling:


#include <iostream>

int main() {
	int b = 10;
	int *a = &b;
	std::cout << "The address location of b = " << &b << std::endl;
	std::cout << "The content at the pointer a = "<< a << std::endl;
	std::cout << "a points to the content at b = "<< *a << std::endl;
	return 0;
}

Let's Learn About Memory

Ask permission from the computer to allocate memory space to store a datatype (in this case int). Name the allocated memory b.

int b;

By assigning a value to b, we store the integer 10 at the memory location we call b.

b = 10;

Ask the computer for permission to read the content at b. Next, add the content to the integer 1. Then, I am asking for the summation to be assigned to b. Notice that 1 is not a double like 1.0. Since b only has enough memory for integer values, adding other datatypes like std::string, double, or long int can cause unpredictable errors.

b = b + 1;

When you ask the computer to read at the memory location named b, you are accessing a numerical address in hexadecimal. To call the serial number location of your variable b use the ampersand symbol, &:

std::cout << "The address location of b = " << &b << std::endl;

To call the hexadecimal address in numbers only:

std::cout << (long int)(&b) << std::endl;

The Mutator Problem

An accessor only accesses or "gets" the variables. That's why accessors are oftentimes known as get functions. Thus, the variables remain constant.


#include <iostream>

int getvar(int content) {
	std::cout << (long int) &content << std::endl;
	return content;
}

int main() {
	int b = 10;
	b = b + 1;

	std::cout << (long int) &b << std::endl;
	std::cout << getvar(b) << std::endl;
	return 0;
}
